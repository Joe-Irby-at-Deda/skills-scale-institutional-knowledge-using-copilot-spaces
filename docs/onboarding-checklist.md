# Onboarding Checklist (Project-level)

Purpose: Ensure new joiners have the required access, knowledge, and tasks to be productive quickly.

Pre-first day:
- [ ] Add new joiner to project roster
- [ ] Create workspace / access request for necessary systems
- [ ] Assign buddy and Onboarding Champion

First day:
- [ ] Environment setup (dev environment, tool access)
- [ ] Introductory meeting with Project Manager and Tech Lead
- [ ] Walkthrough of docs/ and role responsibilities
- [ ] Setup for code review and CI access

First week:
- [ ] Complete small starter task and PR
- [ ] Review architecture and design docs with Tech Lead
- [ ] Complete any required compliance training
- [ ] Check-in with Onboarding Champion to close any open items

Ongoing:
- [ ] Pair programming sessions scheduled
- [ ] Mentorship/buddy check-ins
- [ ] Documentation Owner confirms onboarding docs are accurate

Notes:
- Onboarding Champion and Documentation Owner co-own this checklist and track completion.
